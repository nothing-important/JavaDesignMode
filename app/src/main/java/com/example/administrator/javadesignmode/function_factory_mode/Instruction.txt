优点：
1.基于工厂角色和产品角色的多态化设计是工厂方法模式的关键，工厂可以自主确定创建何种产品对象，并且创建过程隐藏。
2.当系统增加新产品的时候，无须修改抽象工厂和抽象产品，客户端，各种具体产品也不需要修改，只需要增加新的具体产品和具体工厂即可，
增加系统可扩展性，完全符合“开闭原则”，优于简单工厂模式
缺点：
1.在增加新产品时，需要增加对应具体产品和具体工厂，系统中类的个数成对数增加，在一定程度上增加了系统的复杂程度，产品越多，
系统运行需要编译的类也越多，会增加开销。
2.由于考虑到系统的可扩展性，需要引入抽象层，在客户端代码均使用抽象层进行定义，增加了系统的抽象性和理解难度，且在实现时可能需要
DOM、反射等技术，增加了系统的实现难度
